# Metodologia

Esta seção descreve a organização para execução das tarefas do projeto e as ferramentas utilizadas para manutenção do código e demais artefatos.

## Gerenciamento de Projeto

A metodologia ágil escolhida para o desenvolvimento deste projeto foi o Scrum, pois como citam Amaral, Fleury e Isoni (2019, p. 68), seus benefícios são a "visão clara dos resultados a entregar; ritmo e disciplina necessários à execução; definição de papéis e responsabilidades; empoderamento dos membros da equipe para atingir o desafio; conhecimento distribuído e compartilhado de forma colaborativa."

## Processo

A equipe utiliza o GitHub Projects para o acompanhamento do desenvolvimento, organizado em etapas (sprints). As tarefas são atualizadas conforme seu andamento nas seguintes colunas:

- **Backlog:** representa o Product Backlog e recebe todas as tarefas do projeto a serem trabalhadas.
- **To Do:** representa o Sprint Backlog da etapa atual, com as tarefas definidas no início de cada sprint.
- **In Progress:** lista das tarefas que foram iniciadas.
- **Ready to Test:** tarefas finalizadas e prontas para revisão ou teste.
- **Testing:** tarefas em processo de revisão e/ou teste.
- **Done:** tarefas concluídas, revisadas e prontas para entrega.
